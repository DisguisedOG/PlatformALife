# AGENTS.md — Agent guidance for PlatformALife

Purpose: provide short, actionable instructions and key repo facts so AI coding agents can be productive immediately.

Quick start
- Build C# code (from repo root):

```powershell
dotnet build
```

- Run the game with a Mono-enabled Godot editor (C# autoloads require Mono):

```powershell
& 'C:\path\to\Godot_v4.6.3-stable_win64_mono.exe' --path 'D:\KiwiKing Studios\PlatformALife' --scene 'res://Scenes/World.tscn'
```

- Regenerate tileset resource (headless Godot script):

```powershell
& 'C:\path\to\Godot.exe' --path . --script build_tileset.gd
```

Key facts & common pitfalls
- Godot: project uses Godot 4.6.3 (stable). Use a Mono-enabled editor to run scenes that include C# (`Scripts/*.cs`). Non-Mono editors will print `No loader found for resource: res://Scripts/*.cs`.
- C# project: `PlatformALife.csproj` targets `net8.0`. Run `dotnet build` in the repository root to compile.
- Tileset: `Assets/AorakiAtlas.png` is 160×128 → a 5×4 grid of 32×32 tiles. TileSet is generated by `build_tileset.gd` to `Assets/AorakiTileSet.tres` — prefer regenerating rather than hand-editing the `.tres` file.
- If you edit `.tres` or atlas files externally, clear Godot import caches: delete matching files in `.godot/imported/` and `.godot/editor/` and then restart the editor.
- There is a runtime validator `Scripts/AssetValidator.cs` that checks tile/portal placement and collider sizes. Use it to enforce runtime asset rules.
- `Scripts/WaystoneNavigation.cs` was excluded from the project previously due to containing non-C# content; see `PlatformALife.csproj` for exclusions.

Agent conventions (concise)
- Always run `dotnet build` before launching a Mono Godot session for testing C# changes.
- If an automated change touches tile atlases or `.tres` files, run `build_tileset.gd` and clear `.godot` caches before reporting "tile atlas" errors.
- Prefer linking to existing docs (see `PlatformALife_Build_Manual.md` and `Docs/`) rather than duplicating long content.
- When proposing code changes that affect runtime resources, include a short verification plan (build + run scene with Mono Godot, note what to check in the scene).
